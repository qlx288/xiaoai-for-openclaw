# Superpowers 💪

专业软件开发工作流，让 AI 遵循标准开发流程。

## 14 种开发技能

| 技能 | 用途 |
|------|------|
| 🧠 **brainstorming** | 头脑风暴，设计前讨论 |
| 📋 **writing-plans** | 制定开发计划 |
| 🧪 **test-driven-development** | TDD 测试驱动开发 |
| 🔧 **systematic-debugging** | 系统性调试 |
| 👀 **verification-before-completion** | 完成前验证 |
| 🔍 **receiving-code-review** | 接受代码审查 |
| 📝 **requesting-code-review** | 请求代码审查 |
| 📦 **subagent-driven-development** | 子任务开发 |
| ⚡ **dispatching-parallel-agents** | 并行任务分发 |
| 🌿 **using-git-worktrees** | Git 工作树使用 |
| 🔀 **finishing-a-development-branch** | 完成开发分支 |
| 📚 **writing-skills** | 编写新技能 |
| 📖 **using-superpowers** | 如何使用技能 |

## 核心原则

1. **不能跳过设计** - 必须先头脑风暴
2. **必须写测试** - TDD 流程
3. **必须验证** - 完成后必须有证据
4. **不能跳步骤** - 遵循开发流程

## 使用场景

当用户说：
- "帮我写个功能" → 先头脑风暴
- "这个 bug 怎么修" → 用系统性调试
- "代码写完了" → 必须验证
- "帮我审查代码" → 用代码审查流程
